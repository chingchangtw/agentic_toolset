Dual-Track Orchestration

ts-orchestrate routes every work type across two parallel tracks — Discovery validates ideas, Delivery ships epics. Source: src/skills/ts-orchestrate/SKILL.md

Orchestrator ts-orchestrate — session entry point Reads [WORKFLOW STATE] hook context · never reads raw state files

Commands

  • :start entry — set active_epic + dial, route
  • :status cross-layer Discovery + Delivery view
  • :next enforced phase advance + gate checks

Entry gate

  • Delivery-entry needs an active epic. If none →
  • /ts-project:plan --new
  • /ts-discover idea "<desc>"
  • POC & unvalidated FEATURE exempt (Discovery-entry)

Security gates

G1threat-model — feature, refactor, epic · blocks Think→Plan
G2sec-review — epic, ops, security patch · blocks Ship
▲ never auto-signed at any autonomy level
routes to one or both tracks →
Discovery Validate ideas

Event-storming + DDD validation before code exists. Entry: /ts-discover idea. Idea graduates to the backlog only when validated.

  • 1idea

    Run /ts-discover explore <id> · WIP limit 3 in exploring+validating

  • 2exploring

    ts-event-storming-facilitator output required to exit → validate (mandatory if any H-risk) or decide build

  • 3validating

    council-advisor + tows-strategy-analyst + ts-ddd-tactical-validator → /ts-discover decide <id>

  • 4ready

    Run /ts-project plan --sync — move epic(s) into the backlog → hands to Delivery

Decide outcomes / side states keep-learning → re-explore; at 3× forced decide killed ADR written, kept for audit reduce-scope split into new ideas
Delivery Ship epics

Per-epic phase spine driven by /ts-deliver:refine. Spine varies by work type; the full 7-phase spine below is the epic/feature path.

  • 1thinkG1

    Spectra:discuss + G1 threat-model sign-off (if required) → refine

  • 2plan

    Spectra:propose + design review → refine

  • 3build

    Spectra:apply + test coverage gate → refine

  • 4review

    staff-review report → refine

  • 5test

    acceptance + integration gates → refine

  • 6shipG2

    Spectra:archive + G2 sec-review sign-off (if required) → refine

  • 7reflect

    /ts-iteration:next (or :close if last epic) · spike writes learning entry first

Spine varies by work type bugfix / hotfix Think→Build→Ship chore Build→Ship patch Build→Test→Ship spike Think→Build→Reflect ops Think→Build→Review→Ship
↻

Feedback loop. On epic completion (Reflect), mark status: done in iteration.json, write an entry to discovery.json with source_epic, then /ts-iteration:next. Discovery resumes with the completed epic as context — the two tracks feed each other.

Workflow Routing

Keyed by the work type the end-user wants. The route names its track(s) inline.

Work typeRoute
Feature Discidea → explore → validate (H-risk) → decide build → plan --sync → DelivThink → Plan[G1] → Build → Review → Test → Ship. Already validated? Enter at /ts-iteration:next.
Bugfix & Hotfix DelivThink → Build → Ship (no gates; HOTFIX = distinct audit tag, expedited)
Refactor DelivThink → Plan[G1] → Build → Review → Ship → Reflect
Chore & Tech debt DelivBuild → Ship (no gates, no Think; large tech debt → route as Refactor)
Patch DelivBuild → Test → Ship (G2 sec-review when security-related)
Spike DelivThink → Build → Reflect → Disclearning entry written to discovery.json (no Ship)
PoC Disc onlyidea → explore → validate (throwaway prototype) → decide build/kill/reduce-scope (never ships)
Ops / Infra DelivThink → Build → Review → Ship[G2] (infra touches the security surface)
Epic Plan slice, not a work type. Reached via /ts-iteration:next on a synced epic: Think → Plan[G1] → Build → Review → Test → Ship[G2] → Reflect
