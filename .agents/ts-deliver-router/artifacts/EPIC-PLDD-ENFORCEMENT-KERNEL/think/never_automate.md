# Never Automate — PLDD Enforcement Kernel

- Never activate PLDD in `agenticToolset` without Dogfood Activation approval.
- Never edit root dependency/configuration files during Phase A.
- Never modify live `.agents/*`, hooks, router, or registry as implementation output.
- Never treat unsupported import syntax as compliant.
- Never weaken or inline-disable a layer rule to make a check green.
- Never replace legacy Discovery arrays with object arrays.
- Never allow host adapters to define unique architecture or contract rules.
- Never sign G1/G2 or Dogfood Activation on behalf of human.
